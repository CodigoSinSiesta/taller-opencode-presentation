# Slide 11 — Buenas prácticas / alucinaciones

## Tesis
El agente genera texto probable, no verdad. La verificación es responsabilidad humana.

## Datos
~30% del código generado puede contener errores sutiles (paquetes inexistentes, APIs mal).

## Haz / Evita
- Haz: contexto explícito, criterios de éxito, ejecutar tests tú, iterar pequeño.
- Evita: copiar APIs sin verificar, aprobar el plan a ciegas, sobrecargar tareas.

## Frase ancla (Horthy 2026)
"1.000 líneas de plan ≈ 1.000 de código." Lee el código del diff, no solo el plan.

## Transición
"Y el cambio de fondo: el repo como prompt persistente — harness engineering."
