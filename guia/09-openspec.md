# Slide 9 — OpenSpec (SDD)

## Tesis
OpenSpec añade una capa ligera de specs: acordamos el qué antes del cómo, y queda versionado.

## Puntos clave
- `openspec/`: project.md (contexto), specs/ (vigentes), changes/ (en curso) + archive/.
- El change es revisable como un PR; la spec viva es contexto para el próximo agente.
- Dentro de OpenCode vía comandos `/opsx:*`.
- Instalado con `openspec init --tools opencode` (ya hecho en el starter).

## Contraste
Vibe coding = directo al código. Spec-driven = acuerdo explícito y trazable.
